## 1. Mapeamento do Processo

### Fluxo Atual da Equipe

![Fluxo da equipe](/docs/atividade-09/fluxo.png)

O processo é um loop: quando os testes encontram defeitos, o fluxo retorna à etapa de desenvolvimento/correção antes de seguir para a entrega.

## 2. Entradas, Atividades e Saídas

| Etapa | Entrada | Atividade | Saída |
|---------|---------|---------|---------|
| Recebimento da demanda | Requisito | Análise da tarefa | Tarefa definida |
| Desenvolvimento | Requisito | Implementação | Código desenvolvido |
| Testes | Código | Execução de testes (manuais e automatizados) | Evidências e defeitos |
| Correções | Defeitos encontrados e registrados | Ajustes no sistema | Nova versão |
| Retorna para o desenvolvimento | Ajuste do requisito | Retoma o desenvolvimento até corrigir | Desenvolvimento |
| Entrega | Versão validada | Publicação | Funcionalidade entregue |

## 3. Reflexão sobre o Processo

### 1. O processo utilizado pela equipe está claramente definido?
O MVP foi feito as pressas, visando funcionalidade acima de tudo. Portanto, o foco da equipe estava no desenvolvimento das funcionalidades chaves da aplicação, porém deixando de lado a organização dos processos.

### 2. Todos os integrantes seguem o mesmo fluxo de trabalho?
Sim, porém falta organização e estrutura de equipe para que isso aconteça.

### 3. Em quais etapas a qualidade é verificada?
Durante os testes somente.

### 4. Quais melhorias poderiam tornar o processo mais eficiente?
- Melhor organização do papel de cada colaborador.
- Planejamento das tarefas com Trello.
- Code Review
- Gerenciamento do projeto pelo Github
- Utilização de Pull Request para melhor acompanhamento das tarefas
- Controle definido de mudanças.
- Estrutura mais organizada do fluxo de trabalho.

### 5. Como a qualidade do processo impacta a qualidade do produto final?
A aplicação teve inicial foco na funcionalidade, deixando de lado o processo.
O processo foca em como o software é desenvolvido, principalmente nas atividades e prevenção dos defeitos. Isso gera uma melhor produtividade. Isso impacta principalmente na experiência do usuário ao utilizar a aplicação.
